# Unit-03 ビジネスルール

**作成日**: 2026-04-16
**対象ストーリー**: US-T-020（感情タグ記録）・US-T-030（感情傾向グラフ）

---

## BR-U03-01: タグ型制約

| ルール | 内容 |
|---|---|
| BR-U03-01a | `type = 'emotion'` のタグには `category` が必須（positive/negative/neutral） |
| BR-U03-01b | `type = 'context'` のタグには `category` は NULL |
| BR-U03-01c | DB 制約（CHECK）でアプリ層バイパスを防止 |

---

## BR-U03-02: システムデフォルトタグ

| ルール | 内容 |
|---|---|
| BR-U03-02a | テナント作成時に感情タグ 15個 + コンテキストタグ 8個を自動シード |
| BR-U03-02b | `is_system_default = true` のタグは教員が削除できない |
| BR-U03-02c | 教員はカスタムタグを追加できる（既存機能維持） |
| BR-U03-02d | カスタムタグ作成時は `type` の指定が必須 |
| BR-U03-02e | 既存テナントへのマイグレーション: 新タグを `ON CONFLICT DO NOTHING` で挿入 |

---

## BR-U03-03: 感情傾向集計

| ルール | 内容 |
|---|---|
| BR-U03-03a | 集計対象は `type = 'emotion'` のタグのみ（context タグは除外） |
| BR-U03-03b | 1エントリに複数の感情タグがある場合、それぞれをカウント（例: 喜び + 安心 → positive +2） |
| BR-U03-03c | 集計単位は日（DATE(created_at)、Asia/Tokyo） |
| BR-U03-03d | 本人のエントリのみ集計（RLS + user_id WHERE 二重防御） |

---

## BR-U03-04: 期間切り替え

| ルール | 内容 |
|---|---|
| BR-U03-04a | 期間は `week`（7日）・`month`（30日）・`quarter`（90日）の3段階 |
| BR-U03-04b | 起点は常に「今日」（サーバー時刻 Asia/Tokyo） |
| BR-U03-04c | デフォルト期間は `week` |
| BR-U03-04d | 不正な period 値は 400 エラー |

---

## BR-U03-05: グラフ表示判定

| ルール | 内容 |
|---|---|
| BR-U03-05a | 期間内のエントリ数が 3件未満の場合、グラフを非表示にする |
| BR-U03-05b | 非表示時はガイドメッセージを表示: 「記録を続けるとグラフが表示されます（あと N 件）」 |
| BR-U03-05c | 判定は `totalEntries`（感情タグ付きエントリ数）ではなく、期間内の全エントリ数で行う |

---

## BR-U03-06: アクセス制御

| ルール | 内容 |
|---|---|
| BR-U03-06a | `/api/private/dashboard/*` は認証必須（401） |
| BR-U03-06b | 教員は自分のデータのみ閲覧可能（他教員のデータは RLS で自動除外） |
| BR-U03-06c | 管理者ロールは本 API を使用しない（管理者向けは Unit-04 で別途設計） |

---

## BR-U03-07: フロントエンド表示

| ルール | 内容 |
|---|---|
| BR-U03-07a | `/journal` ページ上部にサマリーカード（直近の感情傾向の簡易表示） |
| BR-U03-07b | `/dashboard/teacher` に詳細グラフ（Recharts 折れ線グラフ + 期間セレクター） |
| BR-U03-07c | データがない日はグラフ上で 0 として表示（折れ線が途切れないように） |
| BR-U03-07d | positive=緑系・negative=赤系・neutral=灰系で色分け |
