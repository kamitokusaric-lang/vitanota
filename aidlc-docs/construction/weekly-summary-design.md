# 週次レポート (今週のひとこと) 機能設計

- **Status**: [CURRENT] 設計確定 (実装未着手)
- **作成日**: 2026-04-27
- **作成者**: chimo + Claude

---

## 1. 目的

教員が日々ノートの投稿をもとに、**1 週間を無理なく振り返れるようにすること**。

- 評価しない
- 分析しすぎない
- 安心して書ける状態を保つ

## 2. 設計思想 (最重要・踏み絵)

> AI は「見抜く存在」ではなく、「そっと言い換える存在」。
> AI は「分析」ではなく「**ねぎらい**」を行う。

- 個人を評価しない
- 観測されてる感をゼロに保つ (= memory `feedback_observed_moment_broken.md` の踏み絵を死守)
- 投稿の安心感を最優先

「励まし」ではなく「ねぎらい」。励ましは押すが、ねぎらいは受け止める。

## 3. 機能概要

- 日々ノートの投稿 + タスク状況をもとに、AI が週次の「ねぎらいの言葉」を生成
- 表示対象は **本人のみ** (school_admin にも他教員にも見えない)
- 表示名: **今週のひとこと**
- ダッシュボード大タブの 1 つとして配置 (旧 `weekly` disabled タブを active 化)

## 4. 生成タイミング

- **その週の初回アクセス時に自動生成**
- 同じ週の 2 回目以降のアクセスは保存済みを返す (再生成しない)
- 1 ユーザー × 1 週 = 1 回の生成
- 「週」の定義: **月曜始まり** (月曜〜日曜の 7 日間、`week_start` = 月曜の日付)

## 5. データフロー

```
[投稿時]
① POST /api/private/journal/entries
   ↓
② 前処理 (正規表現マスキング)
   ↓
③ DB 保存 (content + content_masked 両方)

[週次レポート要求時]
④ GET /api/me/weekly-summary
   ↓
⑤ 今週分の summaries レコードあり? → あれば返却 (短絡)
   ↓ なければ
⑥ 今週分の集計 SQL 実行
   ↓
⑦ AI 呼出し (Haiku 4.5、1 段階)
   ↓
⑧ ガードレールチェック (人称・数字・長さ)
   ↓
⑨ DB 保存 + 返却 (本人のみ)
```

## 6. マスキング (投稿時の前処理)

### 6.1 方針

- 投稿時に **正規表現マスキング** を実行
- 完璧でなくて OK の方針 (思想優先)
- LLM 不使用 (コスト 0、レイテンシ 0)
- DB 保存時点で `content_masked` カラムに格納
- 本人表示は `content` (原文)、AI 入力は `content_masked` のみ使用

### 6.2 マスキングルール

| 対象 | 置換先 |
|---|---|
| ○○くん / 君 / ちゃん | 生徒 |
| ○○先生 / 教諭 / 主任 / 校長 / 教頭 | 同僚 |
| ○○さん / 様 | ある人 |
| ○年○組 / 数字組 | クラス |
| ○○小学校 / 中学校 / 高校 / 学園 | 学校 |

### 6.3 正規表現セット

```typescript
// src/features/journal/lib/mask-content.ts
const MASK_PATTERNS: Array<{ regex: RegExp; replace: string }> = [
  // 児童・生徒系
  { regex: /[一-龯ぁ-んァ-ヶー]{1,4}(?:くん|君|ちゃん)/g, replace: '生徒' },

  // 教員系
  { regex: /[一-龯ぁ-んァ-ヶー]{1,4}(?:先生|教諭|主任|校長|教頭)/g, replace: '同僚' },

  // 一般敬称 (保護者・関係者)
  { regex: /[一-龯ぁ-んァ-ヶー]{1,4}(?:さん|様)/g, replace: 'ある人' },

  // クラス名
  { regex: /\d+年\d+組(?:\d+班)?/g, replace: 'クラス' },
  { regex: /[一二三四五六]年[一二三四五六]組/g, replace: 'クラス' },
  { regex: /\d+組/g, replace: 'クラス' },

  // 学校名
  { regex: /[一-龯ぁ-んァ-ヶー]{1,8}(?:小学校|中学校|高等学校|高校|学園)/g, replace: '学校' },
];

export function maskContent(text: string): string {
  return MASK_PATTERNS.reduce(
    (acc, { regex, replace }) => acc.replace(regex, replace),
    text,
  );
}
```

### 6.4 実装場所

- `src/features/journal/lib/mask-content.ts`
- POST `/api/private/journal/entries` および PUT 編集時に呼び出し

### 6.5 過剰検出について (許容範囲)

「○○さん」のマスキングで以下のような一般語が引っかかる可能性:
- 「子どもさん」 → 「ある人」
- 「皆さん」 → 「ある人」
- 「お客さん」 → 「ある人」

→ chimo の判断で許容 (個人特定回避を優先、原文は本人にだけ見せるので影響限定)

## 7. 集計データ (AI 入力)

週次レポート生成時に、以下を集計して AI に渡す。

```
今週 (M月D日 月〜M月D日 日) の振り返り素材:

【投稿】
- 総投稿数: N 件
- mood 分布: 😊 a 件 / 🙂 b 件 / 😐 c 件 / 😥 d 件 / 😣 e 件
- 主なタグ: 気づき (3), 疲れ (2), 工夫 (1)
- 公開設定: 公開 X 件 / 非公開 Y 件

【タスク】
- 完了: N 件 (今週中に done になった、updatedAt 近似)
- 動きあり: M 件 (今週中に新規作成 or 更新、updatedAt が今週内)
- 期限到来: K 件 (dueDate が今週内)
- 依頼を受けた: J 件 (新規 / createdBy ≠ 自分 & assignee = 自分)
- 他人へ依頼した: L 件 (新規 / createdBy = 自分 & assignee ≠ 自分)
```

集計の正確性メモ:
- 「完了」は `status='done' AND updatedAt が今週内` で近似。厳密な status 変更履歴は MVP 後で対応。
- 全集計は scope='mine' (自分が assignee or createdBy) のタスクが対象。

## 8. AI プロンプト (1 段階統合版)

### 8.1 プロンプト本体

```
あなたは、ある学校の教員の今週の記録を見て、そっとねぎらいの言葉をかけるアシスタントです。

【今週の記録】
- 期間: {YYYY/MM/DD}(月) 〜 {YYYY/MM/DD}(日)

【投稿 (マスク済み)】
{投稿リスト、1 投稿 1 行}

【集計】
- 投稿数: {N} 件 / 公開 {x} / 非公開 {y}
- mood 分布: 😊{a} / 🙂{b} / 😐{c} / 😥{d} / 😣{e}
- 主なタグ: {tag1} ({n1}), {tag2} ({n2}), ...
- タスク完了: {N} / 動きあり: {M} / 期限到来: {K} / 依頼を受けた: {J} / 他人へ依頼: {L}

---

【内部処理 (出力には書かない)】
1. 投稿と集計から、今週の流れを「対人 / 業務 / 内省 / 感情 / 工夫 / 疲労」等の抽象的な傾向として把握する
2. その傾向に対して、そっとねぎらいの言葉を生成する

【出力ルール (絶対)】
1. 個人を評価しない (✗「あなたは頑張りました」)
2. 原文を引用しない、言葉を真似ない
3. マスク後の語 (「生徒」「同僚」「ある人」「クラス」「学校」) も文中に出さない
4. 数字や統計を文章に持ち込まない (✗「8 件投稿しました」)
5. 断定を避ける (✗「忙しい一週間でした」 → ○「歩いてきた一週間だったようです」)
6. アドバイスや指示をしない
7. 問題指摘ゼロ、見守る姿勢
8. 一日や瞬間ではなく、一週間全体にそっと声をかける
9. 励ましじゃなく、認める / 受け止める / そばにいる
10. 投稿が少ない週でも、書かなかったことを問題視しない

【トーン】
- やさしい / 控えめ / 静か / 見守る
- 語彙: 「お疲れさま」「ひと息」「無理のない」「そのまま」「歩いてきた」「重ねてきた」
- 避ける: 「素晴らしい」「凄い」「頑張って」「~するべき」「分析」「成長」

【長さ・形式】
- 200〜300 文字、3〜4 段落、段落間は空行で余白
- 「今週のひとこと」として 1 つだけ出力 (見出しや前置き不要)
```

### 8.2 サーバー側 fallback ガードレール

AI 出力後、以下を check:

| Check | 動作 |
|---|---|
| 出力に「あなた」「君」「○○先生」等の人称 | 「先生」に置換 (or fallback テンプレ) |
| 出力に数字 (件数等) | warning ログ |
| 400 文字超 | 自動切り詰め |
| 50 文字未満 | fallback テンプレ表示 |

### 8.3 fallback テンプレート (生成失敗時 / ガード違反時)

```
今週も、一週間お疲れさまでした。

書いたこと、手を動かしたこと、その全部がここにあります。

少し、ひと息ついてくださいね。
```

## 9. データ保存

### 9.1 新規テーブル

```sql
CREATE TABLE journal_weekly_summaries (
  user_id      uuid        NOT NULL,
  week_start   date        NOT NULL,  -- 月曜日の日付
  summary      text        NOT NULL,
  generated_at timestamptz NOT NULL DEFAULT now(),
  PRIMARY KEY (user_id, week_start)
);
```

RLS: `user_id = auth.uid()` で本人のみアクセス可。

### 9.2 既存テーブルへのカラム追加

```sql
ALTER TABLE journal_entries
  ADD COLUMN content_masked text;
```

### 9.3 既存投稿の backfill

migration 適用後、既存 `journal_entries` 全件に対して maskContent を適用する backfill スクリプトを実行。

### 9.4 保存方針 (設計書原文より)

- AI 出力 (`summary`) のみ保存
- 入力データ (集計内容) は保存しない
- 原文 (`content`) と マスク済み (`content_masked`) は journal_entries に共存
- ログに原文を残さない

## 17. Phase 2 (MVP 後の拡張余地)

- 自動週次バッチ (EventBridge + Lambda) — アクセスがなくても月曜 0 時に生成
- 出し分けロジック (ポジティブ多 / 気づき多 / しんどい多 で出力方向を切替)
- LLM ベースのマスキング (より高精度)
- 個別投稿の抽象化を別段階に分離 (より深い「ねぎらい」生成)
- status 変更履歴テーブル (「完了」を厳密に取れる)
