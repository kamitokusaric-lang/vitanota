# vitanota — コスト試算

**作成日**: 2026-04-12  
**前提条件**: 開発者1名・利用校1校・対象教員20名  
**対象スコープ**: MVP（フェーズ1）および月々の運用コスト  

---

## 開発費用（MVP）

### 工数見積もり

| 開発項目 | 見積もり工数 |
|---|---|
| プロジェクト基盤（Next.js・DB・CI/CD・Docker環境） | 35〜40時間 |
| Google認証・ロール管理・マルチテナント・RLS設定 | 45〜55時間 |
| 日誌エントリ CRUD・タイムライン UI | 50〜60時間 |
| 感情スコア・感情カテゴリ記録（日誌と統合） | 20〜25時間 |
| タグ付与機能 | 15〜20時間 |
| 教員ダッシュボード・感情グラフ（Recharts） | 40〜50時間 |
| 管理者ダッシュボード（全教員ステータス一覧・カード） | 55〜70時間 |
| アラートシステム（スケジュール実行・検知・クローズ） | 40〜50時間 |
| セキュリティ実装（HTTPヘッダー・バリデーション・CORS等） | 35〜45時間 |
| テスト実装（ユニット・統合・テナント隔離・E2E） | 80〜100時間 |
| バッファ（修正・レビュー・デバッグ） | 40〜50時間 |
| **合計** | **455〜565時間** |

> **補足**: セキュリティ要件15項目・文科省ガイドライン対応・テナント隔離テストなど、学校向け要件として厳密に設定したため、一般的なWebアプリより工数が20〜30%多くなっています。

## 留意事項

- 本試算は2026年4月時点の目安です。為替レート・クラウドサービスの料金改定により変動します
- 開発者の契約形態（正社員・フリーランス・開発会社）によって単価は大きく異なります
- **1校・20名に対して開発費用が高い**のは、マルチテナント・セキュリティ・テスト計画など「複数校への拡張を前提とした設計」を要件に含めているためです。2校目以降は追加開発費用がほとんど発生しません
- AWS Bedrock の利用にはAWSアカウントとモデルアクセス申請が必要です。初期設定に別途数時間の工数が発生します
- 文科省ガイドラインへの準拠確認・クラウドサービス利用承認手続き（NFR-01-EX-2）に要する調整工数は含んでいません
